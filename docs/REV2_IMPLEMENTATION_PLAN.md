### 🔴 Bug 1: Keyboard covers input in Lease → Contact Tenant → Message (S2-10)

**Client says:** *"Keyboard issue — the keyboard overlaps the typing area"*

**Root cause (confirmed in code):**
- The shared `KeyboardAvoidingView` component at `src/shared/components/layouts/KeyboardAvoidingView.tsx:17` defaults to `behavior='height'` on Android:
  ```tsx
  behavior={behavior || (Platform.OS === 'ios' ? 'padding' : 'height')}
  ```
- `app.config.js` already sets `softwareKeyboardLayoutMode: 'resize'` for Android
- When Android **resizes** the window natively AND `KeyboardAvoidingView` also applies `behavior='height'`, they **conflict** — the keyboard still overlaps or the layout jumps
- PR #10 fixed the explicit callers (`MessageThreadScreen`, `ComposeMessageScreen`) but left the shared default broken. Any screen not explicitly overriding `behavior` still gets the double-adjustment.

**Fix (one line):**
```tsx
// line 17 — change 'height' to undefined
behavior={behavior || (Platform.OS === 'ios' ? 'padding' : undefined)}
```
This lets Android's native `resize` mode handle the keyboard without React Native's `KeyboardAvoidingView` also trying to adjust.

**Files to change:**
- `src/shared/components/layouts/KeyboardAvoidingView.tsx` — line 17 only

**Verification:** Android emulator — open Lease → Contact Tenant → Message, tap input field, verify keyboard does not overlap. Check that other text inputs across the app (payments, profile, maintenance, messages) also work correctly.

### 🔴 Bug 2: Tenant lease PDF not visible / downloadable (S2-24)

**Client says:** *"The downloadable lease was not able to be seen"*

**Root cause:**
- `TenantLeaseScreen.tsx` only shows the "Download PDF" button when `lease.lease_document_url` is non-null
- The PDF is generated by the **owner** via `OwnerLeaseDetailScreen`'s "Generate PDF" button (calls `regenerateLeasePDF`)
- If the owner never clicked that button, `lease_document_url` stays null → tenant sees "Generating document..." but no download option
- There is **no auto-generation** when the lease becomes `active`

**Fix:**
1. **Auto-generate** the lease PDF when lease status transitions to `active` (after both parties sign — add call in the signature save handler)
2. Add a **"Generate PDF" action button** in `TenantLeaseScreen.tsx` when `lease_document_url` is null but the lease is already `active`/signed — so the tenant doesn't depend on the owner

**Files to change:**
- `src/features/tenant/screens/TenantLeaseScreen.tsx` — add "Generate PDF" when document missing + lease active
- `src/features/owner/screens/OwnerLeaseDetailScreen.tsx` — add auto-generation on lease activation
- `src/features/leases/api/regenerateLeasePDF.ts` — verify it can be called safely from tenant context

**Verification:** Create/sign a lease as both tenant and owner → verify PDF is auto-generated and downloadable from the tenant dashboard.

